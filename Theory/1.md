## 📝 Detailed Notes

### 1. Importance of System Design

- System Design is the most critical part of technical interviews at major companies (MAANG) like Microsoft, Google, Facebook, and Amazon.
- It is essential for final-year students, third-year students, and working professionals looking to advance their careers.
- 🎯 **Interview Tip:** The instructor notes that during his own interview at Optum, core System Design questions were asked.

### 2. Defining System Design

- **Formal Definition:** The process of defining the architecture, components, modules, interfaces, and data for a system.
- **Simplified Explanation:** It is the planning phase of building software, apps, or websites. It involves creating a "blueprint" of how everything will work together.
- 💡 **Analogy:** If you want to build WhatsApp, you first plan the architecture for chats, status updates, calls, and user management before touching the technical implementation.

### 3. Why System Design is Crucial

- **Scalability:** Ensuring the system can handle an increasing number of users and data without failing.
- **Performance:** Improving speed so that actions (like sending a message) happen instantly rather than taking minutes.
- **Reliability:** Ensuring the system remains functional even if a component fails; having backup plans.
- **Maintainability (Ease of Update):** Designing the system so new features (like WhatsApp "Communities") can be added without causing problems for existing users.
- **Cost and Time Efficiency:** Managing resources so the project doesn't become overly expensive or time-consuming.
- ⚠️ **Warning:** If you don't plan your database (DB) bandwidth and limits, the "system will burst" when traffic spikes (e.g., millions of messages per second on WhatsApp).

### 4. Types of System Design (HLD vs. LLD)

- **High-Level Design (HLD):** Focuses on major components and feature-oriented planning (e.g., "I need a chat app where User A sends an encrypted message to User B via a DB").
- **Low-Level Design (LLD):** Focuses on technical implementation, including class diagrams, function signatures, and internal logic.
- **LLD Example for WhatsApp Message:** A `Message` class containing `UserID`, `From`, `To`, `Timestamp`, `MessageContent`, and `ReadStatus` (Sent, Delivered, Seen).

### 5. Key Components of System Design

- **Architecture Design:** How different parts (Client/Server/DB) interact.
- **Database Design:** Deciding between SQL and NoSQL; determining where to store images and videos.
- **API Design:** Creating endpoints for login, profile, and messages that work across different platforms (Web vs. Mobile).
- **Load Balancing:** Distributing traffic across different servers so the system doesn't crash during peak times.
  - 💡 **Example:** During festivals like Diwali, traffic surges; load balancers prevent the system from "bursting".
- **Caching:** Storing frequently used data (like a user profile) locally or in a fast-access layer to reduce slow database calls.
- **Data Sharding:** Dividing data to improve performance when dealing with millions of records.
- **Redundancy & Replication:** Ensuring data availability even during server failures.
- **Security:** Using algorithms to ensure end-to-end encryption for messages.

### 6. Real-World Example: Instagram

- **Major Components:** User Service, Auth (Login/Signup), Feed Service, Comments Service.
- **Storage:** Using AWS S3 for photos and a CDN (Content Delivery Network) for fast photo fetching.
- **Databases:** Using Relational DBs for user data and NoSQL for feeds and images.

### 7. Step-by-Step System Design Process

1. **Requirement Gathering:** Defining functional requirements (calls, messages) and non-functional requirements (speed, number of users).
2. **System Interfaces:** Designing how the system interacts with users.
3. **Architecture:** Choosing between Monolithic or Microservices.
4. **Database Selection:** Choosing SQL/NoSQL based on data type (e.g., S3 for images).
5. **API Design:** Defining how Front-end and Back-end communicate.
6. **Scalability & Fault Tolerance:** Planning for growth and failures.
7. **Security:** Ensuring data doesn't leak.
8. **Review & Optimization:** Final check of the documentation and design.

### 8. Conceptual Analogy: Designing a City

- **Software System = City Layout:** Planning where buildings and roads go.
- **APIs = Roads:** Connecting different parts of the city (Police station, Mall, Library).
- **Load Balancer = Traffic Police:** Directing people to different roads to prevent traffic jams.
- **Database = Library/Bank:** Storing all the information or money (data).
- **Cache = Local Newspaper Stand:** Providing a quick, general overview of information so people don't have to go to the main library every time.

## 🔁 Comparisons & Trade-offs

| Feature             | High-Level Design (HLD)                        | Low-Level Design (LLD)                        |
| ------------------- | ---------------------------------------------- | --------------------------------------------- |
| **Focus**           | Major components and overall architecture      | Detailed technical implementation             |
| **Orientation**     | Feature-oriented                               | Tech/Class-oriented                           |
| **Example**         | Designing social media flow, DB choice         | Class diagrams, function signatures, logic    |
| **Interview Stage** | Usually discussed first to show overall vision | Discussed later to show deep technical skills |

## 🎯 Interview Questions from This Video

1. **What is System Design, and why is it important for modern applications?**
   - _Answer:_ System Design is the process of defining architecture and components to build a scalable, reliable, and efficient system. It ensures that an app can handle millions of users and high performance without failing.
2. **What is the difference between HLD and LLD?**
   - _Answer:_ HLD focuses on the "big picture" (major components like DBs and servers), while LLD focuses on the "minute details" (classes, methods, and specific code logic).
3. **How does a Load Balancer improve system reliability?**
   - _Answer:_ It distributes incoming traffic across multiple servers, ensuring no single server is overwhelmed, which prevents system crashes during peak traffic (like festivals).
4. **Why is Caching used in systems like WhatsApp or Instagram?**
   - _Answer:_ Caching stores frequently accessed data (like profile pictures) locally or in a fast layer to avoid repeated, slow calls to the main database, thereby improving performance.
5. **What are the key steps in the system design process?**
   - _Answer:_ Gathering requirements, defining interfaces, choosing architecture, selecting databases, designing APIs, and reviewing for scalability and security.

## 📌 Key Takeaways

- System Design is a **blueprint** for software that covers architecture, modules, and data.
- A successful design must be **Scalable, Reliable, and Robust**.
- **HLD** is for the big picture; **LLD** is for the technical details.
- Real-world design requires balancing **Functional** (features) and **Non-functional** (speed/users) requirements.
- Analogies like **city planning** help simplify complex architectural components like APIs and Load Balancers.

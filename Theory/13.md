## 📝 Detailed Notes

### Introduction to CDN & Global Load Delivery

- Shivansh introduces Episode 13, shifting from core database concepts to global delivery strategies.
- **CDN (Content Delivery Network):** A system of distributed servers that deliver content based on the geographic locations of the user.

### The Geography of Latency

- **Physical Distance Matters:** Data traveling from a server in the USA to a user in India is inherently slower than data from a local server in Mumbai or Delhi.
- **The File Size Factor:** While small files (1-2 KB) show negligible delay, massive files like a **5 GB video** suffer significant lag over long distances due to "network hops."

### Edge Servers & CDN Mechanics

- **Edge Server:** The server physically nearest to the user.
- **Two Primary Approaches:**
  1.  **Reactive (Pull):** The first user to request data experience a delay while the Edge server pulls it from the Origin. Subsequent users get it instantly from the Edge cache.
  2.  **Proactive (Push/Sync):** Systems sync data to all global Edge servers "overnight" or as soon as content is uploaded, so the very first user finds it ready.

### CDN Flow & Cache Management

- **The Cycle:** User requests file $\rightarrow$ Edge server checks for local copy $\rightarrow$ **Cache Hit** (serve immediately) or **Cache Miss** (pull from Origin, cache it, then serve).
- **Handling Stale Content:** When a file is updated, the cache becomes "stale."
  - **Versioning:** Use tags like `v1`, `v2`, or `v3` to force the CDN to recognize and cache the latest version.
  - **Multi-checks:** Implementing logic to verify if the cached data is still fresh (referencing concepts from Episode 09).

### Benefits of Using a CDN

- **Performance:** Faster page loads and lower latency.
- **Cost Efficiency:** Reduces the load on the Origin server, saving on compute and bandwidth costs.
- **DDoS Protection:** Because requests hit the distributed Edge servers first, malicious traffic spikes are absorbed by the CDN rather than crashing the main backend.

### Core CDN Architecture Components

- **Origin Server:** The main backend where the "master copy" of data is first uploaded (e.g., a new YouTube video).
- **Edge Server:** Local servers distributed worldwide to handle user requests.
- **DNS Routing:** The intelligent pathing that directs a user's request to the correct Edge server.
- **Cache Layer:** The storage within Edge servers specifically for holding large files.

### Global Load Balancing (GLB)

- **Definition:** Distributing **dynamic traffic** across multiple data centers in different countries/regions.
- **Goal:** Fast page rendering and server response globally, not just static file delivery.
- **Customized Responses:** GLB allows you to serve different content based on the user:
  - **Language:** Direct Indian users to servers with Hindi content and US users to English servers.
  - **Region:** Route European users to Frankfurt and Asian users to Mumbai/Singapore.
- **Routing Criteria:** Can be based on **Location, Language, Region (Asia-Pacific), or IP Address.**

### Four Types of Global Load Balancing

1.  **Geo-based:** Routes based purely on the user's physical location.
2.  **Latency-based:** Routes to the server that provides the fastest response (e.g., a powerful 10 GB RAM machine over a 2 GB one).
3.  **Health Check-based:** If the nearest server is down, traffic is automatically diverted to a healthy server in another country to maintain **Availability**.
4.  **Anycast Routing:** Multiple servers share the same IP; the one topologically "closest" responds.

```javascript
// Flow for fetching a large file (Video/Image) via CDN
function getFile(fileName) {
  if (edgeCache.exists(fileName)) {
    // Cache Hit: Serve immediately from local Edge server
    return edgeCache.get(fileName);
  } else {
    // Cache Miss: Fetch from Origin, cache locally for others, then serve
    const fileData = originServer.fetch(fileName);
    edgeCache.store(fileName, fileData);
    return fileData;
  }
}
```

## 🔁 Comparisons & Trade-offs

| Feature             | CDN (Content Delivery Network)          | GLB (Global Load Balancing)                        |
| :------------------ | :-------------------------------------- | :------------------------------------------------- |
| **Primary Content** | **Static Files** (Images, Videos, PDFs) | **Dynamic Traffic** (API requests, Page rendering) |
| **Main Mechanism**  | **Caching** at the Edge                 | **Routing** to the best Data Center                |
| **Primary Goal**    | Reduce file download latency            | Manage global traffic & ensure availability        |
| **Protection**      | Absorbs DDoS spikes at the edge         | Provides failover if a whole region goes down      |

## 🎯 Interview Questions

1.  **Why do we use CDNs primarily for large files?**
    Small files (1-2 KB) travel fast enough that the overhead of a CDN isn't always justified, but for large files (5 GB+), geographical distance creates massive, noticeable delays.
2.  **How does a CDN provide DDoS protection?**
    It acts as a distributed shield; malicious traffic hits many global Edge servers instead of overwhelming the single Origin server.
3.  **What is the difference between Geo-based and Latency-based GLB?**
    Geo-based routes purely by location; Latency-based routes to the server that is currently performing the fastest, which might not always be the closest one if that server is overloaded.
4.  **Explain "Anycast Routing."**
    It is a method where multiple servers share the same IP address, and the network automatically routes the request to the nearest available node.
5.  **How can you customize user experience using GLB?**
    By routing based on language or IP, you can serve a Hindi version of your site to Indian users and an English version to US users from their respective local servers.

## 📌 Key Takeaways

- **CDNs** are for **files** (Static); **GLB** is for **traffic** (Dynamic).
- Always define your **Origin** and **Edge** server locations during design.
- Use **Versioning** to solve the "Stale Cache" problem.
- Global systems should combine **GLB + CDN** for maximum performance and DDoS resilience.
- **Health Checks** are critical in GLB to ensure high availability across continents.
